Python 1.1 to 1.3

开始学习Python

1.1 开始学习

Python最开始由Guido van
Rossum创立，并在1991发布，是一种广泛使用的高级程序语言。Python语言有以下优点：动态类型系统；自动内存管理；支持多种程序样式主要包括：面向对象，命令式和函数式编程，过程化等。它拥有一个强大复杂的标准图书库。
目前经常使用的两个版本：

  • 3.x版本是目前版本，在积极的开发中
  • 2.x版本是之前的版本，截止到2020年，只进行安全性方面的更新，不再添加新的功能。虽然很多项目仍然在使用2.x版本，但迁移到3.x版本是很容易的

你可以在这里下载并安装任意一个版本。比较下两个版本的不同。另外，一些第三方会提供Python的重新打包版本，会附带一些常用的图书库和其它的功能，例如数学，数据分析，科学上的应用等。在官方网站上查看这些列表。

检查Python是否已经安装

为了确认你的Python是否正确安装，你可以在你最爱的终端上输入以下指令来确认（如果你是Windows系统，你需要提前为Python设置好环境变量，之后在cmd窗口输入下面的指令）：
$ python --version

Python 3.x Version ≥ 3.0

如果你已经安装了Python3版本，并且这个就是你的默认版本（你可以在Troubleshooting了解更多细节） 你应该看到以上内容；

$ python --version

Python 3.6.0
Python 2.x Version ≤ 2.7

如果你已经安装了Python2版本，并且这个就是你的默认版本（你可以在Troubleshooting了解更多细节） 你应该看到以上内容；
$ python --version

Python 2.7.13

如果你已经安装了Python3版本，但是输出却是2.x的版本，说明你也安装了Python2版本。这种情况经常在MacOS和Linux分布式系统上出现，你可以用$ python3代替。

Hello World Python文件

创建一个叫做hello.py文件，里面包含以下内容：
对于Python 3.x Version ≥ 3.0版本：
print('Hello, World')
对于Python 2.x Version ≥ 2.6版本，你可以通过导入future模块里的来使用Python 3.x Version ≥ 3.0版本里的print功能：

from __future__ import print_function

Python 2有很多可选功能可以从Python 3中导入使用。
在Python 2版本里，你也可以像下面这样写，但这个在Python 3里是不合法的，因此不推荐使用因为它减弱了跨版本代码兼容性。

print 'Hello, World'

在你的终端，找到hello.py文件。
输入python hello.py，然后按下Enter键：

$ python hello.py

Hello, World

你应该看到Hello, World输出在控制台上。
你也可以用文件路径替换$ python hello.py ，比如你文件放在Linux里的home/user/hello.py，你也可以写python /home/user/hello.py。

1.2 创建变量并赋值

在Python中创建变量，你需要为变量具体一个名字并赋值。
<variable name> = <value>

python用=来给变量赋值，不需要在变量前面声明该变量类型，该变量类型与赋值的类型一致，变量必须要初始化。

# Integer
a = 2   
print(a)
# Output: 2

# Integer
b = 9223372036854775807    
print(b)   
#Output: 9223372036854775807

  # Float
pi = 3.14
print(pi)
#Output: 3.14

# String
c = 'A'  
print(c)    
# Output: A

# String
name = 'John Doe'
print(name)
# Output: John Doe


# Boolean
q = True
print(q)
# Output: True


# NULL
x = None
print(x)
# Output: None

变量赋值从左到右工作,下面是一个错误的例子：

0 = x
=> Output: SyntaxError: can't assign to literal

不能用关键字作为变量名，你可以用下面代码查看关键字：

import keyword  
print(keyword.kwlist)

变量命名规则：

  1. 变量名首字母必须为字母或下划线：
    x = True # valid
    _y = True # valid
    9x = False # starts with numeral
    => SyntaxError: invalid syntax
    $y = False # starts with symbol
    => SyntaxError: invalid syntax
  2. 剩下部分由字母、数字或下划线组成：
    has_0_in_it = "Still Valid"

  3. 后面调用的变量在前面必须有定义：

    x = 9
    y = X*5
    =>NameError: name ‘X’ is not defined

即使在Python中声明变量时也不需要指定数据类型，而分配变量内存中的必要区域，Python解释器自动选择最合适的内置类型： 

a = 2
print(type(a))
# Output: <type 'int'>
b = 9223372036854775807
print(type(b))
# Output: <type 'int'>
pi = 3.14
print(type(pi))
# Output: <type 'float'>
c = 'A'
print(type(c))
# Output: <type 'str'>
name = 'John Doe'
print(type(name))
# Output: <type 'str'>
q = True
print(type(q))
# Output: <type 'bool'>
x = None
print(type(x))
# Output: <type 'NoneType'>

当使用=执行赋值操作时，左边的=右边的对象的名称。最后，将右边的对象的引用分配给左边的名称。想当于：左边是指针，指向右边起始地址，如果是变量，指向给该变量分配的内存地址。
a_name = an_object # "a_name" is now a name for the reference to the object "an_object"

因此，从上面的许多赋值例子中，如果我们选择pi＝3.14，那么pi是一个名字（不是名字，因为对象3.14可以有多个名称。）如果你不理解下面的内容，回到这一点。再看一遍！此外，你可以看看这个，以便更好地理解。

可以将多个值赋给一行中的多个变量。注意，两边个数要一样： 

a, b, c = 1, 2, 3
print(a, b, c)
# Output: 1 2 3
a, b, c = 1, 2
=> Traceback (most recent call last):
=> File "name.py", line N, in <module>
=> a, b, c = 1, 2
=> ValueError: need more than 2 values to unpack
a, b = 1, 2, 3
=> Traceback (most recent call last):
=> File "name.py", line N, in <module>
=> a, b = 1, 2, 3
=> ValueError: too many values to unpack

最后一个例子可以在左边多加任意一个名字来避免错误，一般这种哑变量可以用任意名字，但常用下划线-代替：
a, b, _ = 1, 2, 3
print(a, b) 

# Output: 1, 2

你也可以把一个值同时赋值给多个变量：

a = b = c = 1
print(a, b, c)

使用这种赋值法，要注意：这三个名字指向同一个内存地址，在之后改变其中一个变量值，并不会对其他两个变量带来影响：

a = b = c = 1 # all three names a, b and c refer to same int object with value 1
print(a, b, c)
# Output: 1 1 1
b = 2 # b now refers to another int object, one with a value of 2
print(a, b, c)
# Output: 1 2 1 *so output is as expected.*

以上也适用于可变类型（如list、dict等），就像不可变类型（如int、String一样）：
x = y = [7, 8, 9] `# x and y refer to the same list object just created, [7, 8, 9]x = [13, 8, 9]# x now refers to a different list object just created, [13, 8, 9]` `print(y) # y still refers to the list it was first assigned# Output: [7, 8, 9]`

到现在为止，一直都还不错。在修改对象时，情况有点不同（与分配名称相反）当级联赋值用于可变类型时，我们执行了一个不同的对象。看一看
下面，你将首先看到它：
x = y = [7, 8, 9]
# x and y are two different names for the same list object just created, [7,8, 9]
x[0] = 13
# we are updating the value of the list [7, 8, 9] through one of its names, xin this case
print(y) # printing the value of the list using its other name
# Output: [13, 8, 9] # hence, naturally the change is reflected

嵌套列表在Python也是合法的，这意味着列表元素可以为列表：
x = [1, 2, [3, 4, 5], 6, 7] # this is nested list
print x[2]
# Output: [3, 4, 5]
print x[2][1]
# Output: 4

最后，变量名的类型是可以改变的，自动变成右边赋值的类型：
a = 2
print(a)
# Output: 2
a = "New value"
print(a)
# Output: New value

相当于左边指针，右边分配内存，指针自动指向该内存，类型与右边值一致。

1.3 代码块缩进

Python使用缩进来定义控制和循环结构。这会让程序具有可读性，但是程序员要注意空格的使用。

Python使用冒号和缩进来表明模块的开始和结束，其它语言不一定这样。Python中的模块像函数、循环、分支等其他结构是没有结束判定符的，全是用冒号开始加上下面缩进的内容：

def my_function(): # This is a function definition. Note the colon (:)
    a = 2 # This line belongs to the function because it's indented
    return a # This line also belongs to the same function
print(my_function()) # This line is OUTSIDE the function block

or

if a > b: # If block starts here
    print(a) # This is part of the if block
else: # else must be at the same level as if
    print(b) # This line is part of the else block

包含一个单行语句的块可以放在同一行上，但一般不推荐使用：

if a > b: print(a)
else: print(b)

如果不是一个语句块这样写就会出现错误：

if x > y: y = x
print(y) # IndentationError: unexpected indent
if x > y: while y != z: y -= 1 # SyntaxError: invalid syntax

空块也会导致错误，当模块内容为空时用关键字pass代替：

def will_be_implemented_later():
    pass

Spaces vs. Tabs

通常用四个空格缩进。
使用Tabs缩进也是可以的除了PEP 8,它更喜欢空格缩进。
Python 3.x Version ≥ 3.0
Python 3.x不允许这两者混用，在这种情况下，编译时错误生成：在缩进中不一致地使用制表符和空格，程序将不运行。
Python 2.x Version ≤ 2.7
Python 2.x允许混用，但这非常不提倡使用。 \t缩进是八个Spaces倍数。虽然有的编辑器会把Tab配置成4个Spaces，但可能会出现微小的错误。

PEP 8:
在使用-t选项调用Python 2命令行解释器时，它会发出有关代码的警告。
非法混合标签和空间。当使用-TT时，这些警告会变成错误。这些选项是强烈推荐！

许多编辑器都有“制表空间”配置。在配置编辑器时，应该区分字符（\t）和Tab键。

  • \t应该被配置为显示8个空格，以匹配语言语义。编辑器还可以自动将制表符\t转换为Spaces。
  • 但是，配置编辑器可能会有助于按下Tab键将插入4个空格，而不是插入\t字符。
